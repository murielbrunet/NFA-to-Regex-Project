What is a nondeterministic finite automata?

A NFA is a nondeterministic machine, which means several choices may exist for the next state at any point. To not get confused with DFAs (deterministic finite automatas), here is the formal definition of a NFA:

  • Q : finite set of states
  • Σ : alphabet
  • q0 : start state
  • F : set of accepting/final states
  • δ : transition function: δ : Q x Σε → P(Q) [power set], including ε, and mapping to a set of states (including ∅)

    • The NFA is accepted if there exists some path through the NFA to an accepting state (almost as if the NFA guesses the path).

    What is a regular expression?

    In arithmetic, we can use the operations + and x to build up expressions such as (5+3) x 4. Similarly, we can use the regular operations to build up expressions describing languages, which are called regular expressions. Here's an example: (0U1)0*.

    The formal definition of a regex is:

    Say that R is a regular expression if R is
  • a for some a in the alphabet Σ,
  • ε,
  • ∅ (empty set),
  • (R1 U R2), where R1 and R2 are regular expressions,
  • (R1 o R2), where R1 and R2 are regular expressions, or
  • (R1*), where R1 is a regular expression.

    • How are they related?

    Regular expressions and finite automata are equivalent in their descriptive power. Any regular expression can be converted into a finite automaton that recognizes the language it describes, and vice versa.
